5. PRÓXIMOS PASOS → OE2
================================================================================

Con las bandas B04 y B08 almacenadas en BD, el OE2 puede proceder a:

OE2 - APLICAR CÁLCULO DE ÍNDICES ESPECTRALES (NDVI)
  1. Leer bandas desde BD (ya disponibles gracias a OE1)
  2. Calcular NDVI = (B08 - B04) / (B08 + B04)
  3. Normalizar valores a rango [-1, 1]
  4. Clasificar salud: Critical / Alert / Healthy
  5. Generar visualización coloreada (rojo/amarillo/verde)
  6. Permitir comparación temporal (evolución del cultivo)

ARCHIVOS A CREAR (OE2):
  - backend/app/services/ndvi_service.py
  - frontend/app/components/organisms/NDVIVisualization.tsx
  - backend/tests/test_ndvi_service.py

NOTA: El prototipo de NDVI ya existe en notebook, solo requiere migración
      a servicio FastAPI e integración con frontend.

6. LECCIONES APRENDIDAS (DOCUMENTADAS EN tasks/lessons.md)
================================================================================

✅ Docker-compose es crítico para validación end-to-end
   → Tests pasando NO garantiza sistema completo funcionando

✅ Nubosidad en zona tropical requiere criterio ajustado
   → 20% max (vs 10-15% estándar Europa) para tener fechas suficientes

✅ Fechas inteligentes mejoran UX
   → -6 meses desde hoy, primer día del mes, bloquear fechas futuras

✅ Responsive requiere diseños duales
   → Mobile: bottom sheet con overlay
   → Desktop: sidebar con backdrop opcional

✅ STAC API agrupa múltiples tiles por fecha
   → Necesario deduplicar y quedarse con menor nubosidad
